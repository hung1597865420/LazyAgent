# Agent Harness - Báo cáo Review Code tự động

## Kết luận: **🟢 APPROVE**

### Tóm tắt:
Panel findings are mostly medium/low edge cases around staged-file handling plus a few data-integrity risks in the new coordination flows. No critical/high blocker was identified, but the medium issues should be addressed to avoid false commit blocks, missed review coverage, or stale coordination state.

## Chi tiết các Findings

| install_hooks.py | 3487 | MEDIUM | edge_case | A staged path containing a newline can make the pre-commit hook fail closed or review the wrong paths if staged files are collected with newline-delimited output. If `_staged_file_context()` / coordination record collection splits `git diff --cached --name-only` with `splitlines()`, the single path becomes multiple paths; subsequent staged blob reads can fail and `run_hook()` exits 1. | Use NUL-delimited Git output everywhere staged paths are enumerated, e.g. `git diff --cached --name-only -z` and split on `b'\0'`; pass paths to Git via `--` and argv list, never shell/string joins. Add a test that stages a file named `"a\nb.py"` and verifies the hook completes and the context contains the exact path. | tester |
| install_hooks.py | 3487 | MEDIUM | edge_case | Deleting a staged file can block commits if `_staged_file_context()` blindly reads every staged path as a blob. For deleted paths, index blob reads such as `git show :old.py` fail because there is no staged blob; with fail-closed `run_hook()` behavior, a valid deletion-only commit can exit 1. | When enumerating staged files, include status from `git diff --cached --name-status -z` and skip blob reads for status `D` while still passing deletion metadata to coordination/review. Add a test for a deletion-only commit and a mixed modify+delete commit. | tester |
| install_hooks.py | 3487 | MEDIUM | assumption | UTF-16/UTF-32 text source files are treated as binary because NUL-byte detection runs before decoding. A UTF-16LE Python file contains frequent `\x00` bytes, so `_staged_file_context()` can classify it as binary and omit reviewable content, allowing source changes to bypass pre-commit review semantics. | Do not classify solely by presence of NUL for known text/code extensions; try BOM-aware decoding first (`utf-16`, `utf-32`) or use `tokenize.detect_encoding`/charset detection for source-like files before falling back to binary. Add a test with a UTF-16LE file with BOM and verify content is included or the hook explicitly blocks unsupported encoded source files. | tester |
| auto_watch.py | 1497 | MEDIUM | partial_failure_gap | `record_file_event(...)` is called before triggering automation, but the event recording and the subsequent `auto_trigger(...)` execution are not described as one atomic operation. If event recording succeeds and the trigger fails, coordination state can claim or imply work was observed/handled even though the actual processing did not run. | Record a lifecycle event with explicit status transitions, e.g. `queued` then `triggered`/`failed`, or persist the event and trigger result in a single operation/result table entry. Ensure failures update coordination state so later gates can distinguish observed-but-unprocessed changes. | integrity |
| tools/runner.py | 10061 | MEDIUM | partial_failure_gap | Goal runner coordination leases are released only when the run reaches the normal finish path. If an exception, process termination, or early failure occurs after heartbeat/claim but before `release_files(...)`, leases can remain active until stale expiry and unnecessarily block other sessions. | Wrap runner execution in `try/finally` and call `release_files(...)` for any leases owned by the session on all exit paths. Also record an operation result status such as `failed` or `aborted` so stale cleanup can distinguish crashed runs from active runs. | integrity |
| mcp_server.py | 4203 | MEDIUM | partial_failure_gap | `call_tool()` automatically sends `session_heartbeat(task=f"mcp:{tool_name}")` before each tool call. If argument validation or the tool execution fails before corresponding cleanup/result recording, the session may look active for a task that never actually ran, which can skew active-session/conflict decisions until heartbeat expiry. | Record tool lifecycle state explicitly: heartbeat/session activity plus per-tool `started`, `failed`, and `completed` operation results. On validation failure, either avoid task-specific heartbeat updates or immediately record a failed/no-op result. | integrity |
| runtime_flags.py | 5029 | MEDIUM | data_integrity | `active_workspace_root()` now prioritizes `WORKSPACE_ROOT` over `CLAUDE_PROJECT_DIR` and metadata. If `WORKSPACE_ROOT` is stale or inherited across shells, coordination workspace scoping can be derived for the wrong repository, causing cross-repo lease pollution, false conflicts, or missed conflicts. | Validate that `WORKSPACE_ROOT` exists and contains the current project/git root before using it for coordination identity. If it disagrees with the detected repository root, either fail closed with a clear error or prefer the detected root. Add tests for stale/inherited `WORKSPACE_ROOT` values. | integrity |
| harness-full-setup.bat | 2134 | MEDIUM | partial_failure_gap | Setup performs multiple persistent actions such as profile writes, feature toggles, startup/watch configuration, and now coordination-related state, but there is no described transaction or compensation plan across the whole setup flow. A failure midway can leave profiles/features/watch tasks partially updated. | Add a setup transaction journal or checkpoint file recording each persistent step and its previous state, then provide rollback/repair on failure. At minimum, emit a final consistency check and actionable remediation when any later step fails after earlier persistent writes succeeded. | integrity |
| install_hooks.py | 3487 | LOW | edge_case | Staged submodule/gitlink updates can fail closed if treated like normal blobs. The index entry mode is `160000`; attempting to read it as file content may error or return non-file data, causing the hook to exit 1 under the new fail-closed path. | Detect index mode/status for staged entries. For mode `160000`, skip blob-content extraction and provide metadata like `submodule path old_sha -> new_sha` to review/coordination. Add a test that stages a submodule pointer change and verifies the hook does not crash. | tester |
