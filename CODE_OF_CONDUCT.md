# Code of Conduct

Agent Harness should be a practical and respectful engineering space.

## Expected Behavior

- Be direct, constructive, and specific.
- Assume good intent while still naming risks clearly.
- Respect maintainers' time by providing reproduction steps, logs, and context.
- Avoid sharing private credentials, endpoints, cost reports, or local machine
  paths in public issues.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing secrets, private data, or other people's credentials.
- Repeated low-effort reports without reproduction details.
- Deliberately bypassing safety, profile, or secret-handling rules in shared
  workflows.

## Reporting

For security-sensitive reports, follow `SECURITY.md` instead of opening a public
issue. For conduct concerns, contact the repository owner privately with the
relevant context and links.
